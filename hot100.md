## 1. 回溯 (Backtracking) - 8题

**统一策略：DFS 前后仅判断当前节点是否能够扩展，扩展后的逻辑由递归开头的剪枝决定。**

这样做的好处是：将合法性检查统一放到递归入口处，循环体内只负责尝试扩展，逻辑更清晰、代码更统一。

```python
def dfs(...):
    # 递归开头：剪枝（判断当前状态是否合法）
    if not isValid(path): return
    if isComplete(path): result.append(...); return

    # 循环体：仅尝试扩展
    for choice in choices:
        path.append(choice)
        dfs(...)
        path.pop()
```

## 7. 哈希表 (Hashing) - 4题

| 状态 | 题号 | 题目 | 难度 |
|------|------|------|------|
| [ ] | 1 | [两数之和](https://leetcode.cn/problems/two-sum/) | 简单 |
| [ ] | 49 | [字母异位词分组](https://leetcode.cn/problems/group-anagrams/) | 中等 |
| [ ] | 128 | [最长连续序列](https://leetcode.cn/problems/longest-consecutive-sequence/) | 中等 |
| [ ] | 560 | [和为 K 的子数组](https://leetcode.cn/problems/subarray-sum-equals-k/) | 中等 |

**为什么滑动窗口在这里失效？**

滑动窗口的本质是**单调性假设**——窗口扩大时求和只会增大，缩小时只会减小。基于这个单调性，才能在 right 指针右移后，安全地收缩 left 指针来逼近目标。

但这道题数组元素可正可负可零：

- 窗口扩大，sum **不一定**增大（加了个负数，反而缩小了）
- 窗口缩小，sum **不一定**减小（丢了个负数，反而变大了）

这意味着不存在一个"左边界只往右移就能保证最优"的单调区间。可能存在一个很长的子数组中间掺杂负数恰好等于 K，而滑动窗口在中间某步因为 sum 超过 K 就提前收缩了 left，永远漏掉这个解。

正确做法：**前缀和 + 哈希表**（O(n)）。

## 常见语法

**any(iterable)** — 只要迭代器中有任意一个元素为真就返回 True，全为假时返回 False。常用于简洁判断列表中是否存在满足条件的元素，如 `any(x == 0 for row in matrix for x in row)` 代替写一个循环。

**链表节点作为 Dict 的 key** — ListNode 默认用内存地址（id）做哈希，同一个节点实例可以正常做 key，但 val 相同的不同节点是不同的 key。做 value 则无任何限制。

**heapq（优先队列/最小堆）**

- 初始化：直接用 list，`heap = []`
- 入堆：`heapq.heappush(heap, val)`
- 建堆：`heapq.heapify(list)`，O(n)
- 弹出最小值：`heapq.heappop(heap)`
- 查看堆顶（不弹出）：`heap[0]`
- 最大堆技巧：存负数，`heapq.heappush(heap, -val)`
- 元组比较：先比第一个元素，相同则比第二个，如 `heapq.heappush(heap, (freq, val))`
